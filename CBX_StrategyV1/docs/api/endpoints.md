# CBX Strategy V1 - API Documentation

## 🔐 Authentication & Security

### POST `/api/v1/auth/login`
Authenticates a user and returns JWT tokens.
- **Security Check**: Enforces minimum 32-character `JWT_SECRET_KEY`.
- **Response**: Sets `access_token` and `refresh_token` in HTTP-only cookies.

### POST `/api/v1/log/frontend-error`
Logs frontend errors to the backend and forwards them to Telegram.
- **Rate Limiting**: 10 errors per minute per client (via Redis).

---

## 📈 Trading & Signals

### GET `/api/v1/signals`
Returns recent breakout signals generated by the Scanner.
- **Data Source**: `breakout_events` table joined with `symbol_registry`.

### GET `/api/v1/trades`
Returns history and current open positions.
- **Metrics**: Includes PnL (USD/R), MFE, MAE, and Hold Bars.

---

## ⚙️ System & Settings

### GET `/health`
Comprehensive health check.
- **Validates**: Database connectivity, Redis status, Binance API permissions, and Telegram Bot status.

### POST `/tg/webhook`
Receives updates from the Telegram Bot.
- **Security**: Requires `X-Telegram-Bot-Api-Secret-Token` header validation.
