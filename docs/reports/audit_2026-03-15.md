## 🔴 Critical Issues (Phải sửa ngay)

### 1. Thiếu file `.gitignore`
- **File:** Root directory
- **Nguy hiểm:** Các thông tin nhạy cảm trong file `.env` (Binance API Key, Telegram Token) có thể vô tình bị đẩy lên GitHub/GitLab. Nếu hacker có bộ key này, họ có thể rút sạch tiền trong tài khoản Binance của bạn.
- **Cách sửa:** Tạo ngay file `.gitignore` và thêm `.env`, `venv/`, `__pycache__/`, `logs/` vào danh sách loại trừ.

### 2. Cấu hình Server không an toàn (Production Loophole)
- **File:** `main.py`
- **Nguy hiểm:** Chạy Uvicorn với `host="0.0.0.0"` và `reload=True`. Chế độ `reload` chỉ dành cho phát triển. Trên server thật, nó làm giảm hiệu năng và tiềm ẩn nguy cơ bảo mật nếu hacker có thể kích hoạt reload bằng cách thay đổi file hệ thống.
- **Cách sửa:** Chuyển `reload=False` khi chạy thực tế và cân nhắc dùng `gunicorn` làm manager.

### 3. Nguy cơ tràn bộ nhớ khi đọc Log (OOM Risk)
- **File:** `main.py` (hàm `get_logs`)
- **Nguy hiểm:** Hệ thống đang đọc toàn bộ file log vào bộ nhớ bằng `f.readlines()`. Nếu file `app_info.log` lớn dần (ví dụ 500MB), mỗi lần bạn mở Dashboard, server sẽ ngốn 500MB RAM. Nếu nhiều người mở cùng lúc, server sẽ sập ngay lập tức.
- **Cách sửa:** Sử dụng kỹ thuật đọc ngược từ cuối file (tail) mà không cần load toàn bộ file vào RAM.

## 🟡 Warnings (Nên sửa)

### 1. API Endpoints không có bảo vệ
- **Triệu chứng:** Các đường dẫn `/api/logs`, `/api/orders`, `/api/ai_brain` không yêu cầu mật khẩu hay Token.
- **Hậu quả:** Bất kỳ ai trong mạng nội bộ hoặc biết IP của bạn đều có thể xem được số dư và chiến lược giao dịch của bạn.
- **Cách sửa:** Thêm một lớp Middleware hoặc API Key đơn giản để bảo vệ các endpoint này.

### 2. Lãng phí Connection (Binance Client)
- **File:** `core/data_ingestion.py`
- **Triệu chứng:** Mỗi lần kéo dữ liệu lại tạo một `AsyncClient` mới và đóng lại.
- **Hậu quả:** Tốn tài nguyên thiết lập kết nối (Handshake) và dễ bị Binance chặn (Rate limit) vì mở quá nhiều connection.
- **Cách sửa:** Sử dụng Singleton pattern để duy trì 1 Connection duy nhất cho toàn bộ app.

### 3. Inline Imports (Code Quality)
- **File:** `main.py`
- **Triệu chứng:** Sử dụng `import` bên trong các hàm xử lý request.
- **Hậu quả:** Code lộn xộn, khó theo dõi dependency và làm chậm tốc độ phản hồi API một chút.
- **Cách sửa:** Đưa tất cả import lên đầu file.
